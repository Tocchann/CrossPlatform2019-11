とは？

Cross-Platform なアプリ開発？

  • Cross-Platform
    → 複数環境

  • アプリ開発
    → エンドユーザーが利用するプログラムの開発

考えておきたい点？

  • プログラムを作る上で考慮しておくべき
    事柄 のうち 特有 なものを
    少しだけ並べてみました

まずは結論！

  • Cross-Platform な
    (アプリ)開発においては
    プロジェクト(プロダクト:製品) の
    ポータビリティ を
    高く維持した状態 で
    開発を継続すること

作る前に考えておくこと

  • 想定する実行環境はまとまっていますか？
    • 複数の環境で動かせる必要があるの？
  • その開発要件に合致する開発環境は整えられますか？
  • その開発要件に合致するテスト環境は整えられますか？

作る前に考えておくこと

  • そのプログラムの 5W1H は？
    • いつ(When)
    • どこで(Where)
    • だれが(Who)
    • 何を(What)
    • なぜ(Why)(その理由は？)
    • どうやって(How)

まずは結論！

  • Cross-Platform な
    (アプリ)開発においては
    プロジェクト(プロダクト:製品) の
    ポータビリティ を
    高く維持した状態 で
    開発を継続すること

ポータビリティとは

  • ポータビリティ: Portability
  • 英辞郎より
    1. 携帯できること、 携帯性、可搬性、軽便
    2. 《コ》 移植性

可搬性を高く

  • 可搬性=状況によらず同じもの
  • 一切 修正(変更)せずに
    複数の実行環境向けに用意できる。

移植性を高く

  • 移植性=状況に応じて変わる(変える)もの
  • 極力 修正(変更)せずに
    複数の実行環境向けに用意できる。

もう少し具体的に！

  • Platform に依存しない共通部分
  • Platform 固有の非共通部分
  • 共通部分と非共通部分の混合部分

Platform に依存しない共通部分

  • 可搬性の高いもの
  • 依存性の低いもの
  • 製品共通プロジェクト

Platform 固有の非共通部分

  • 移植性の低いもの
  • 依存性の高いもの
  • 設定ファイル

共通部分と非共通部分の混合部分

  • 共通部分と非共通部分で分割する
    • インターフェース化する
      • ソース共有
        • 同じソースを参照(リンク共有)
      • プロジェクト共有
        • インターフェースプロジェクト
        • 共通プロジェクトの参照

まずは結論！

  • Cross-Platform な
    (アプリ)開発においては
    プロジェクト(プロダクト:製品) の
    ポータビリティ を

    開発を継続すること

データファイルは？

  • 環境依存を低くする
  • 環境依存を明確にする
  • 環境依存を明示する

バイナリファイルは？

  • ビットレベルの同一化(Endianの影響を受けない)
  • Endian の統一または明示
  • 文字コードの統一または、文字コードの明示

テキストファイルは？

  • 文字コードの統一または、文字コードの明示
  • Endian の統一または明示
  • 改行コードの取り扱いの多様性対応または明示

書式化テキストファイルは？

  • 規格にそって書式化テキストを扱っているか？
  • CSV のような自由度の高い書式の場合は。。。
    • 利用プログラム間で意思統一されているか？
  • JSON で日本語どうやって出力している？
    • UTF8なら。。。になっていない？
